# Vocabulary Telegram Bot (Serverless + Google Sheets)

Интерактивный Serverless Telegram-бот для изучения лексики, который использует **Google Таблицы** в качестве основного интерфейса для управления словарями, а **Firebase Firestore** — для хранения прогресса (SRS алгоритм интервальных повторений).

## Особенности
- **Google Sheets как CMS**: Бот автоматически сканирует вкладки (листы) таблицы и превращает их в категории-темы.
- **Интервальные повторения (SRS)**: Режим Quiz подстраивает интервал показа слова в зависимости от ваших успехов (1 день ➔ 3 дня ➔ 7 дней ➔ x2). При ошибке прогресс слова сбрасывается.
- **Whitelist Авторизация**: Бот полностью игнорирует сообщения пользователей, чьих Telegram ID нет в белом списке базы данных.
- **Бессерверная архитектура**: Работает на базе Firebase Cloud Functions, что обеспечивает практически бесплатную работу и нулевое администрирование сервера.

## 🚀 Инструкция по развертыванию

### Шаг 1: Подготовка окружения
1. Сделайте копию `.env.example` -> `.env` и укажите там свои ключи.
2. Создайте сервисный аккаунт в Google Cloud Console с правами на чтение вашей Google Таблицы.
3. Скачайте JSON-ключ сервисного аккаунта, переименуйте его строго в `google-credentials.json` и положите в корень данного проекта (он не попадет в Git, так как добавлен в `.gitignore`).
4. Поделитесь вашей Google Таблицей с email-адресом этого сервисного аккаунта.

### Шаг 2: Настройка Firebase Cloud
1. Создайте новый проект в [Firebase Console](https://console.firebase.google.com/).
2. **Обязательно** переведите проект на тарифный план **Blaze (Pay as you go)**, так как Cloud Functions работают только на нем (бесплатного лимита более чем достаточно).
3. Перейдите в раздел **Firestore Database** и создайте новую базу данных в режиме *Production Mode* (регион по умолчанию `us-central1` отлично подойдет).

### Шаг 3: Деплой функции
1. Убедитесь, что у вас установлен Node.js (рекомендуется версия 20 или 22).
2. Выполните установку зависимостей:
   ```bash
   npm install
   ```
3. Подключите терминал к вашему проекту Firebase (замените на свой ID проекта из консоли):
   ```bash
   npx firebase-tools use ВАШ_FIREBASE_PROJECT_ID
   ```
4. Запустите развертывание:
   ```bash
   npm run deploy
   ```
   > **Важно!** При самом первом деплое нового проекта Firebase может автоматически включить нужные API (Cloud Build). Из-за этого процесс упадет с ошибкой "failed to create function 1st Gen...". Это нормально! Просто подождите 1 минуту, пока обновятся права доступа Google, и **запустите `npm run deploy` еще раз**.

### Шаг 4: Привязка Webhook и Белый список
Терминал после деплоя выдаст строку `Function URL (botWebhook(us-central1)): https://...` 
1. Привяжите Telegram к вашей серверной функции, введя в адресную строку браузера:
   `https://api.telegram.org/bot<ВАШ_TG_TOKEN>/setWebhook?url=<ВАШ_FUNCTION_URL_ИЗ_ТЕРМИНАЛА>`
2. Добавьте себя в белый список: в базе данных Firestore создайте коллекцию `allowed_users`. Создайте внутри документ так, чтобы его **ID был вашим числовым Telegram ID**. Обязательно добавьте внутрь документа любое условное поле (например `role: admin`), иначе Firebase Console не даст его сохранить.

### Шаг 5: Первая загрузка слов
База данных пока пуста. Функция загрузки слов `syncWordsDaily` настроена на автозапуск каждую ночь. Чтобы стянуть слова прямо сейчас:
1. Зайдите в Google Cloud Console -> **Cloud Scheduler**.
2. Найдите задачу синхронизации `firebase-schedule-syncWordsDaily...`.
3. Поставьте галочку и нажмите **Force Run** (Запустить принудительно).

## Команды бота
- `/start` — инициализация профиля и справка
- `/topic` — выбор активной вкладки словаря (темы)
- `/settings` — настройки направления перевода (RU->ES) и количества слов в квизе
- `/flashcards` — режим бесконечных карточек (приоритет неизученным словам)
- `/quiz` — режим тестирования (SRS) на 4 варианта ответа
